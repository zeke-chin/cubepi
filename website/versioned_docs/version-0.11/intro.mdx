# CubePi

**CubePi** is a Pythonic, async-native agent framework designed for high
performance, readability, and production-grade persistence. It provides a
leaner alternative to graph-based agent runtimes by modeling agent logic as
a linear `while` loop that developers can easily trace and debug.

```bash
pip install cubepi
```

Then jump to the [Quick Start](./getting-started/quick-start) to ship a
working tool-using agent in under five minutes.

## What you get

- **Plain async functions, not graph nodes.** The agent loop is a
  `while` loop over message turns. You build tools as `async def`
  functions; the framework routes calls and executes them in parallel.
- **One streaming pattern.** Every provider yields `StreamEvent`s
  through a `MessageStream`. You iterate it with `async for`. No
  callback registries, no separate handler types for text vs. tools.
- **Append-only checkpointing.** Persistence writes the new messages on
  each turn, not the full transcript. O(1) DB I/O regardless of
  conversation length ‚Äî SQLite for laptops, Postgres for production.
- **Native multi-provider.** Anthropic and OpenAI ship in the box,
  through a `Provider` Protocol. Add a new provider in one class.
- **Nine-hook middleware.** `transform_context`, `convert_to_llm`,
  `transform_system_prompt`, `resolve_tool_call`, `before_tool_call`,
  `after_tool_call`, `should_stop_after_turn`, `after_model_response`,
  `on_run_end` ‚Äî each with explicit composition rules. No mystery node ordering.
- **MCP loaders.** Point at any
  [Model Context Protocol](https://modelcontextprotocol.io) server
  (HTTP or stdio) and get back a list of `AgentTool`s.
- **OpenTelemetry built in.** Attach a `Tracer` and every prompt
  produces a tree of OTel spans aligned with the GenAI Semantic
  Conventions ‚Äî works with Jaeger, Tempo, Honeycomb, Datadog, or any
  OTLP-compatible backend. No payloads recorded by default; opt in
  with `record_content=True` and a `redact` callback.
- **Human-in-the-Loop.** Suspend an agent mid-run, route a decision to
  a human (approve/reject/answer), then resume ‚Äî with full checkpoint
  persistence so the agent survives a process restart while waiting.

## Status

You are reading the **CubePi 0.11** release docs. This is the latest
stable documentation set for installing CubePi and building against the
published 0.11 API.

CubePi 0.11 switches deferred tool groups to the new `dispatch` strategy
by default (prompt-cache-stable expansions via `load_tools` results and
a `deferred_tool_call` dispatcher), adds the `resolve_tool_call`
middleware hook for tool-call rewriting before validation and execution,
and introduces `AgentTool.expose_to_model` for hidden-but-resolvable
tools (see [Changelog](/changelog) for the full list).

Source, issues, and discussion live on
[GitHub](https://github.com/cubeplexai/cubepi).
